Churn-Analysis-Project
This project analyzes customer churn data to understand the factors influencing customer retention and attrition. The analysis identifies key insights, patterns, and trends that can help businesses improve customer satisfaction and reduce churn rates.


The project covers the complete data analysis pipeline, from loading and cleaning the dataset to visualizing insights and building predictive models.
Understand patterns and causes of customer churn
Perform Exploratory Data Analysis (EDA) to extract key insights
Clean and preprocess data for better analysis
Visualize customer behavior and churn model

Name of the dataset is customer churn dataset, and the format is CSV and had included many more columns like :Customer ID - "Unique ID for each customer", MonthlyCharges , Paymentmethod , Total Charges , Churn - "Trget variable indicating if the customer left" ETC...
Done with data cleaning handled for missing null values , converted data types to numerics for example:Total charges and removed duplicates , encoded with categorical variable
Explored the relationship between churn and other variables using:
Univariate analysis: Distribution of churn, gender, tenure, etc.
Bivariate analysis: Churn vs. Contract type, Internet Service, Payment Method

Done with visualizations like:
. Bar plots
. Count plots
. Box plots

Key Insights:
Customers with month-to-month contracts are more likely to churn
Higher monthly charges correlate with higher churn probability
Customers with tech support and online security services tend to stay longer
Long-tenure customers are less likely to leave
